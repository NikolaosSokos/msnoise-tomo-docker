# MSNoise + MSNoise-Tomo — Έτοιμο περιβάλλον Docker (Οδηγός για Windows)

> 🇬🇧 English version: [README.md](README.md)

Ένα **έτοιμο, προ-ρυθμισμένο** περιβάλλον για το [MSNoise](https://www.msnoise.org/)
**μαζί με το πρόσθετο (plugin) [msnoise-tomo](https://msnoise.org/plugins/msnoise-tomo/doc/)
ήδη μεταγλωττισμένο και να λειτουργεί**. Δεν χρειάζεται να παλέψετε με εκδόσεις
Python, παλιές βιβλιοθήκες, μεταγλωττιστές C/C++ ή σφάλματα εγκατάστασης.

Όλα τρέχουν μέσα σε **Docker**. Το μόνο που χρειάζεστε στον υπολογιστή σας είναι
το Docker Desktop — **τίποτα άλλο** (ούτε Python, ούτε conda, ούτε compiler).

---

## 1. Τι είναι αυτό με απλά λόγια

Το MSNoise είναι λογισμικό για παρακολούθηση μεταβολών της σεισμικής ταχύτητας
με χρήση σεισμικού θορύβου περιβάλλοντος. Το `msnoise-tomo` είναι ένα πρόσθετο
που υπολογίζει καμπύλες διασποράς (FTAN) και χάρτες ταχυτήτων (τομογραφία).

Η εγκατάσταση του `msnoise-tomo` «με το χέρι» είναι **διαβόητα δύσκολη** και
συνήθως αποτυγχάνει. Αυτό το πακέτο λύνει το πρόβλημα μια για πάντα: παραδίδει
μια **έτοιμη εικόνα Docker** όπου όλα δουλεύουν, δοκιμασμένα.

---

## 2. Γιατί δεν δούλευε πριν (το πρόβλημα που λύσαμε)

Η εγκατάσταση του plugin σπάει για **πολλούς** λόγους ταυτόχρονα:

- Ο επίσημος τρόπος εγκατάστασης χρησιμοποιεί **Python 3.7 + obspy 1.1.1**, που
  πλέον **δεν χτίζονται** σε σύγχρονα συστήματα.
- Τα παλιά **έτοιμα αρχεία (wheels) δεν υπάρχουν πια** — φιλοξενούνταν στο
  Bintray, που έκλεισε οριστικά.
- Το plugin περιέχει **κώδικα C/C++** που πρέπει να **μεταγλωττιστεί**, άρα
  απαιτεί τα κατάλληλα εργαλεία (compiler, CMake).
- Ο κώδικας κάνει `from flask import Markup`, εντολή που η **σύγχρονη Flask (≥2.3)
  αφαίρεσε**. Έτσι, με νέα Flask, μόλις ενεργοποιείς το plugin **κρασάρει
  ΟΛΟΚΛΗΡΗ η εντολή `msnoise`** — πιθανότατα ακριβώς αυτό αντιμετώπισε ο πελάτης.

Η λύση: «καρφώσαμε» (pin) ένα **συνεκτικό, δοκιμασμένο σύνολο εκδόσεων** (δείτε
[`requirements.lock`](requirements.lock)) και μεταγλωττίζουμε το plugin από μια
**σταθερή έκδοση** του κώδικα μέσα στην εικόνα Docker.

## 3. Προαπαιτούμενα (Windows)

Χρειάζεστε μόνο το **Docker Desktop**:

1. Κατεβάστε το από: https://www.docker.com/products/docker-desktop/
2. Εγκαταστήστε το (αφήστε επιλεγμένο το backend **WSL2**, είναι το προεπιλεγμένο).
3. **Επανεκκινήστε** τον υπολογιστή αν σας το ζητήσει.
4. Ανοίξτε το **Docker Desktop** και περιμένετε να γράψει κάτω αριστερά
   «Engine running» (πράσινο).

Για να ελέγξετε ότι δουλεύει, ανοίξτε το **PowerShell** (πλήκτρο Windows → γράψτε
`PowerShell` → Enter) και τρέξτε:

```powershell
docker --version
docker compose version
```

Αν εμφανιστούν εκδόσεις (π.χ. `Docker version 27...`), είστε έτοιμοι.

> 💡 Όλες οι εντολές παρακάτω γράφονται μέσα στο **PowerShell**.

## 4. Εγκατάσταση (μία φορά)

### Βήμα 1 — Κατεβάστε το project

Αν έχετε **Git**:
```powershell
git clone https://github.com/NikolaosSokos/msnoise-tomo-docker.git
cd msnoise-tomo-docker
```

Αν **δεν** έχετε Git: πατήστε στο GitHub το πράσινο κουμπί **Code → Download ZIP**,
κάντε εξαγωγή (extract) τον φάκελο, και μετά στο PowerShell μπείτε μέσα του:
```powershell
cd C:\Users\<το-όνομά-σας>\Downloads\msnoise-tomo-docker
```

### Βήμα 2 — Φτιάξτε το αρχείο ρυθμίσεων

```powershell
Copy-Item .env.example .env
```

### Βήμα 3 — Κατεβάστε την έτοιμη εικόνα (χωρίς build!)

```powershell
docker compose pull
```

Αυτό **κατεβάζει** την προ-μεταγλωττισμένη εικόνα από το GitHub. Δεν χτίζεται
τίποτα τοπικά — γι' αυτό είναι γρήγορο και δεν «μπλέκεστε» με compilers.

> Αν για κάποιο λόγο θέλετε να την **χτίσετε** μόνοι σας (δεν χρειάζεται κανονικά):
> `docker compose build`

Τελειώσατε! Είστε έτοιμοι να το χρησιμοποιήσετε.

### Εναλλακτικά — αν η σύνδεση είναι αργή ή «κόβεται» (offline εγκατάσταση)

Αν το `docker compose pull` δεν ολοκληρώνεται με τη μία, υπάρχει ένα **ένα και
μοναδικό αρχείο** (~416 MB) που περιέχει ολόκληρη την έτοιμη εικόνα. Κατεβάστε
το με έναν browser/download manager (που συνεχίζει αν κοπεί η σύνδεση) από τον
σύνδεσμο που σας έστειλαν, και μετά:

```powershell
# Στον φάκελο όπου κατεβάσατε το αρχείο:
docker load -i msnoise-tomo-image.tar.gz
```

Μετά συνεχίστε **κανονικά** μέσα στον φάκελο του project — το `docker compose`
θα χρησιμοποιήσει την εικόνα που μόλις φορτώσατε, **χωρίς** να κατεβάσει ξανά
τίποτα:

```powershell
Copy-Item .env.example .env
docker compose run --rm msnoise msnoise info
```

> 💡 Ακόμη κι αν το `docker compose pull` κόβεται, αν το **ξανατρέξετε**
> συνεχίζει από εκεί που έμεινε (τα ολοκληρωμένα κομμάτια δεν ξανακατεβαίνουν).

## 5. Πώς το χρησιμοποιείτε

Υπάρχουν **τρεις** τρόποι. Διαλέγετε όποιον θέλετε — όλοι μοιράζονται τους ίδιους
φακέλους `project\` (το έργο σας + βάση δεδομένων) και `data\` (τα δεδομένα σας).

### Α) Γραμμή εντολών — η κανονική ροή του MSNoise

Βάζετε την εντολή `msnoise` μετά το `docker compose run --rm msnoise`:

```powershell
docker compose run --rm msnoise msnoise info
docker compose run --rm msnoise msnoise p tomo --help
```

> Η **πρώτη** εντολή που τρέχετε αρχικοποιεί αυτόματα τη βάση δεδομένων και
> ενεργοποιεί το plugin tomo. Από εκεί και πέρα το έργο σας ζει στον φάκελο
> `project\` και διατηρείται.

Για διαδραστικό περιβάλλον (να μπείτε «μέσα» στο container):
```powershell
docker compose run --rm msnoise bash
```

### Β) Γραφικό περιβάλλον (Web GUI)

```powershell
docker compose --profile web up
```

Ανοίξτε στον browser: **http://localhost:5000**
Για να σταματήσετε: πατήστε `Ctrl + C` στο PowerShell.

### Γ) JupyterLab (διαδραστική ανάλυση & γραφήματα)

```powershell
docker compose --profile jupyter up
```

Ανοίξτε στον browser: **http://localhost:8888** και δώστε token: `msnoise`
(αλλάζει στο αρχείο `.env`). Το έργο σας είναι στο `/project`.

## 6. Τα δικά σας σεισμικά δεδομένα

Βάλτε τα δεδομένα σας (π.χ. miniSEED / αρχείο SDS) μέσα στον φάκελο **`data\`**.
Μέσα στο container εμφανίζονται (μόνο για ανάγνωση) στη διαδρομή `/data`.

Παράδειγμα σάρωσης αρχείου:
```powershell
docker compose run --rm msnoise msnoise scan_archive --init
```

(Στις ρυθμίσεις του MSNoise δηλώστε τον φάκελο δεδομένων ως `/data`.)

## 7. Προαιρετικά: MariaDB αντί για SQLite

Η **SQLite** (προεπιλογή) είναι ιδανική για να ξεκινήσετε και για τα περισσότερα
έργα. Για **πολύ μεγάλα** δεδομένα ή έντονη παράλληλη επεξεργασία, το MSNoise
προτείνει MySQL/MariaDB. Για να την ενεργοποιήσετε:

1. Στο αρχείο `.env` αλλάξτε τη γραμμή σε:
   ```ini
   DB_TECH=mysql
   ```
2. Προσθέστε `--profile mysql` σε ό,τι τρέχετε, π.χ.:
   ```powershell
   docker compose --profile mysql --profile web up
   ```

Η βάση δημιουργείται αυτόματα και αποθηκεύεται σε Docker volume.

> **Σημείωση:** Η επιλογή βάσης γίνεται την **πρώτη φορά**. Για αλλαγή σε
> υπάρχον έργο, αδειάστε τον φάκελο `project\` και ξεκινήστε ξανά.

## 9. Επίλυση προβλημάτων

| Σύμπτωμα | Λύση |
|---------|------|
| `docker: command not found` ή σφάλμα σύνδεσης | Ανοίξτε το **Docker Desktop** και περιμένετε «Engine running». |
| Η θύρα 5000 ή 8888 χρησιμοποιείται ήδη | Αλλάξτε `WEB_PORT` / `JUPYTER_PORT` στο `.env`. |
| Θέλω να ξεκινήσω από την αρχή | Διαγράψτε τα περιεχόμενα του `project\` και (για MariaDB) `docker compose down -v`. |
| Το κατέβασμα της εικόνας ζητάει login | Δείτε την παρακάτω σημείωση για ιδιωτικό/δημόσιο πακέτο. |

> **Ιδιωτικό αποθετήριο:** αν το GitHub repo είναι ιδιωτικό, η εικόνα μπορεί να
> απαιτεί σύνδεση. Ο διαχειριστής μπορεί να κάνει το πακέτο **δημόσιο** (μία φορά,
> στις ρυθμίσεις του πακέτου στο GitHub) ώστε το `docker compose pull` να δουλεύει
> χωρίς κωδικό.

## 10. Πώς λειτουργεί στο παρασκήνιο (τεχνική εξήγηση)

Για όποιον θέλει να καταλάβει **τι ακριβώς φτιάχτηκε**:

**Η εικόνα Docker (`Dockerfile`).**
Ξεκινάμε από καθαρή Python 3.10. Εγκαθιστούμε τα εργαλεία μεταγλώττισης
(`build-essential`, `cmake`, `ninja`) γιατί το plugin tomo έχει κώδικα C/C++ που
πρέπει να χτιστεί. Στη συνέχεια εγκαθιστούμε **ακριβώς** τις εκδόσεις από το
`requirements.lock`, και τέλος μεταγλωττίζουμε το `msnoise-tomo` από μια
**σταθερή έκδοση (commit)** του πηγαίου κώδικα. Στο τέλος του build γίνεται
αυτόματος έλεγχος ότι όλα φορτώνουν σωστά — αν κάτι δεν ταιριάζει, το build
αποτυγχάνει αμέσως αντί να «σπάσει» αργότερα στον χρήστη.

**Το κλειδί της λύσης.**
Το `requirements.lock` «κλειδώνει» τη **Flask 2.2.5** και **Flask-Admin 1.6.1**.
Αυτό κρατάει διαθέσιμη την εντολή `flask.Markup` που χρειάζεται το plugin —
χωρίς αυτό, το `msnoise` κρασάρει εντελώς μόλις ενεργοποιηθεί το tomo. Επίσης
κρατάμε **numpy 1.26** ώστε ο μεταγλωττισμένος κώδικας C να ταιριάζει με τη numpy
που τρέχει.

**Η αυτόματη αρχικοποίηση (`entrypoint.sh`).**
Κάθε φορά που ξεκινά το container, ελέγχει αν υπάρχει ήδη έργο MSNoise. Αν όχι:
δημιουργεί τη βάση (SQLite ή MySQL ανάλογα με το `.env`) και ενεργοποιεί το
plugin tomo — **μία φορά**, αυτόματα. Έτσι ο χρήστης δεν χρειάζεται να κάνει
χειροκίνητες ρυθμίσεις.

**Οι επιλογές εκτέλεσης (`docker-compose.yml`).**
Με τα «profiles» του Compose διαλέγετε **τι** θα τρέξει: γραμμή εντολών, γραφικό
περιβάλλον, JupyterLab, και SQLite ή MariaDB — χωρίς να αλλάζετε τίποτα στον
κώδικα.

**Ασφάλεια.**
Το γραφικό περιβάλλον και το JupyterLab είναι «δεμένα» μόνο στο `localhost`
(127.0.0.1), δηλαδή προσβάσιμα **μόνο από τον δικό σας υπολογιστή** και όχι από
το τοπικό δίκτυο. (Σημαντικό, γιατί το Jupyter μπορεί να εκτελέσει οποιονδήποτε
κώδικα.)

**Δημοσίευση εικόνας (`.github/workflows/docker-publish.yml`).**
Κάθε φορά που ανεβαίνει αλλαγή στο GitHub, ένα αυτόματο workflow χτίζει την εικόνα
και την ανεβάζει στο GitHub Container Registry. Έτσι εσείς απλώς κάνετε
`docker compose pull` και την παίρνετε έτοιμη.
