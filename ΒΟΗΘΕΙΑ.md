# Οδηγός & Βοήθεια — MSNoise + MSNoise-Tomo (Docker)

> **Αν κόλλησες κάπου:** ανέβασε **αυτό το αρχείο** σε έναν βοηθό τεχνητής
> νοημοσύνης (**Google Gemini** στο gemini.google.com ή **Claude** στο
> claude.ai) και γράψε την ερώτησή σου ή κάνε επικόλληση (paste) το μήνυμα
> σφάλματος. Το αρχείο περιέχει όλο το απαραίτητο πλαίσιο ώστε ο βοηθός να σε
> καθοδηγήσει βήμα-βήμα.

---

## 1. Τι είναι αυτό

Ένα **έτοιμο, πακεταρισμένο περιβάλλον** για το λογισμικό
[MSNoise](https://www.msnoise.org/) μαζί με το πρόσθετο (plugin)
**msnoise-tomo** (τομογραφία επιφανειακών κυμάτων με σεισμικό θόρυβο).

Όλα τρέχουν μέσα σε **Docker**, οπότε **δεν** χρειάζεται να εγκατασταθεί Python,
βιβλιοθήκες ή μεταγλωττιστής στον υπολογιστή — μόνο το πρόγραμμα **Docker
Desktop**.

## 2. Το πρόβλημα που λύθηκε

Η εγκατάσταση του msnoise-tomo «με το χέρι» αποτυγχάνει για πολλούς λόγους μαζί:

- Ο επίσημος τρόπος ζητά **Python 3.7 + obspy 1.1.1**, που πλέον δεν χτίζονται.
- Τα παλιά έτοιμα αρχεία (wheels) **δεν υπάρχουν πια** (έκλεισε το Bintray).
- Το plugin έχει κώδικα **C/C++** που πρέπει να **μεταγλωττιστεί**.
- Ο κώδικας κάνει `from flask import Markup`, εντολή που η **νέα Flask (≥2.3)
  αφαίρεσε** — έτσι, με σύγχρονη Flask, μόλις ενεργοποιηθεί το plugin **κρασάρει
  ολόκληρη η εντολή `msnoise`**.

**Η λύση μας:** φτιάξαμε μια εικόνα Docker που «κλειδώνει» (pin) ένα συνεκτικό,
δοκιμασμένο σύνολο εκδόσεων (κυρίως **Flask 2.2.5**, που κρατάει το `Markup`
διαθέσιμο) και μεταγλωττίζει το plugin από μια **σταθερή έκδοση**. Η εικόνα είναι
δοκιμασμένη ότι δουλεύει.

## 3. Τι φτιάξαμε (περιεχόμενα φακέλου)

| Αρχείο | Ρόλος |
|--------|-------|
| `Dockerfile` | Η «συνταγή» της εικόνας (Python 3.10 + μεταγλώττιση του plugin) |
| `requirements.lock` | Οι **ακριβείς** εκδόσεις όλων των βιβλιοθηκών |
| `docker-compose.yml` | Επιλογές εκτέλεσης (γραμμή εντολών / γραφικό / Jupyter, SQLite / MariaDB) |
| `entrypoint.sh` | Στήνει αυτόματα τη βάση και ενεργοποιεί το plugin την πρώτη φορά |
| `.env.example` | Πρότυπο ρυθμίσεων (αντιγράφεται σε `.env`) |
| `data/` | Εδώ μπαίνουν **τα δικά σου σεισμικά δεδομένα** (φαίνονται ως `/data`) |
| `project/` | Εδώ ζει το έργο σου MSNoise + η βάση (διατηρείται) |
| `README.gr.md` | Αναλυτικός οδηγός στα Ελληνικά |

Η έτοιμη εικόνα είναι δημοσιευμένη εδώ (δημόσια, χωρίς login):
**`ghcr.io/nikolaossokos/msnoise-tomo-docker:latest`**

## 4. Πώς εγκαθίσταται (Windows)

1. **Docker Desktop:** κατέβασέ το από https://www.docker.com/products/docker-desktop/
   και εγκατάστησέ το. Άνοιξέ το και περίμενε «**Engine running**» (πράσινο).
2. **Κατέβασε την εικόνα** — άνοιξε **PowerShell** και τρέξε:
   ```
   docker pull ghcr.io/nikolaossokos/msnoise-tomo-docker:latest
   ```
   (Αν κοπεί, ξανατρέξ' το — συνεχίζει από εκεί που έμεινε.)
3. **Κατέβασε τον φάκελο του project** (κουμπί **Code → Download ZIP** στο GitHub)
   και κάνε **Extract**.
4. **Ρύθμιση + δοκιμή** — στο PowerShell, μέσα στον φάκελο:
   ```
   Copy-Item .env.example .env
   docker compose run --rm msnoise msnoise info
   ```

## 5. Πώς το χρησιμοποιείς

- **Γραμμή εντολών (η κανονική ροή MSNoise):**
  ```
  docker compose run --rm msnoise msnoise <εντολή>
  ```
  π.χ. `docker compose run --rm msnoise msnoise p tomo --help` για το plugin.
- **Γραφικό περιβάλλον (browser):**
  ```
  docker compose --profile web up
  ```
  και άνοιξε **http://localhost:5000** (σταμάτημα: `Ctrl + C`).
- **JupyterLab:**
  ```
  docker compose --profile jupyter up
  ```
  και άνοιξε **http://localhost:8888** (token: `msnoise`).
- **Τα δεδομένα σου:** βάλ' τα στον υποφάκελο `data` → μέσα είναι στο `/data`.

## 6. Συχνά προβλήματα & λύσεις

| Σύμπτωμα | Λύση |
|---------|------|
| `docker: command not found` ή σφάλμα σύνδεσης | Άνοιξε το **Docker Desktop** και περίμενε «Engine running». |
| Το `docker pull` κόβεται | Ξανατρέξε την **ίδια** εντολή· συνεχίζει από εκεί που έμεινε. |
| Η θύρα 5000 ή 8888 «είναι κατειλημμένη» | Άλλαξε `WEB_PORT` / `JUPYTER_PORT` στο αρχείο `.env`. |
| Θέλω καθαρή αρχή | Σβήσε τα περιεχόμενα του φακέλου `project` και ξεκίνα ξανά. |
| `docker compose pull` ζητά login | Σημαίνει ότι το package δεν είναι δημόσιο — πες το στον Νίκο. |

## 7. Πλήρης τεχνική κατάσταση (πλαίσιο για βοηθό AI)

> Αυτή η ενότητα είναι κυρίως για έναν βοηθό τεχνητής νοημοσύνης που θα διαβάσει
> το αρχείο, ώστε να δίνει σωστές απαντήσεις.

- **Λειτουργικό χρήστη:** Windows + Docker Desktop (backend WSL2). Εντολές σε **PowerShell**.
- **Εικόνα Docker (δημόσια):** `ghcr.io/nikolaossokos/msnoise-tomo-docker:latest`
- **Βάση κώδικα:** https://github.com/NikolaosSokos/msnoise-tomo-docker
- **Μέσα στην εικόνα:** Python 3.10, **MSNoise 1.6.5**, **obspy 1.5**, **numpy 1.26**,
  **Flask 2.2.5 / Flask-Admin 1.6.1** (η κρίσιμη διόρθωση που κρατά διαθέσιμο το
  `flask.Markup`), και το **msnoise-tomo** μεταγλωττισμένο από το commit `affcaa4`
  (3 extensions C/C++: `vg_fta`, `mkMatSmoothing`, `mk_MatPaths`).
- **Πώς ορίζεται η βάση:** μεταβλητή `DB_TECH` στο `.env` → `sqlite` (προεπιλογή,
  στον φάκελο `project/`) ή `mysql` (με `--profile mysql`, container MariaDB).
- **Αυτόματη αρχικοποίηση:** το `entrypoint.sh` τρέχει την πρώτη φορά
  `msnoise db init` και `msnoise config set plugins=msnoise_tomo`. Αν υπάρχει ήδη
  `project/db.ini`, δεν ξανα-αρχικοποιεί.
- **Δίκτυο/ασφάλεια:** το γραφικό (5000) και το Jupyter (8888) είναι δεμένα μόνο
  στο `127.0.0.1` (μόνο τοπικά).
- **Τι ΔΕΝ πρέπει να προτείνεις:** εγκατάσταση MSNoise/plugin εκτός Docker (με
  conda/pip στο σύστημα) — αυτό ακριβώς αποτυγχάνει· όλα γίνονται μέσα στο
  container μέσω `docker compose`.
